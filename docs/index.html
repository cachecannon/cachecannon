The Question

How many requests can your server actually handle?

Every production Valkey deployment has a throughput ceiling — the point where tail latency spikes and SLOs break. Knowing that number before you ship is the difference between capacity planning and firefighting at 2 AM.

The built-in valkey-benchmark can't answer this question. It runs one command type at a time, has no rate control, and gives you max-throughput numbers that tell you nothing about behavior at your actual production load.

Mixed workloads

Real traffic is GETs and SETs together. valkey-lab runs them concurrently at your chosen ratio so you see how they contend.

Rate control

Set a target request rate to measure latency at a specific load point — not just flat-out max throughput.

Saturation search

Automatically ramp load until your SLO breaks. One command finds the exact ceiling.

How It Works

Five phases, zero guesswork

Every benchmark run follows the same disciplined sequence. Phases are visible in the output so you always know where you are.

1. Precheck

Verify the server is reachable before committing to a long run.

2. Prefill

Optionally write every key so GETs start from a warm cache.

3. Warmup

Run the workload without recording. Let caches and JIT stabilize.

4. Running

Record per-second samples — throughput, latency percentiles, errors, hit rate.

5. Results

Final summary with GET/SET latency breakdowns and bandwidth.

Saturation Search

Find the ceiling automatically

Define your latency SLO. valkey-lab starts at a low request rate, increases by 5% each step, and stops after three consecutive failures. The result is the maximum throughput that stays within your SLO. 

valkey-lab saturate --slo-p999 1ms -c 16 -P 32

Get Started

Install and run

# build from source (requires Linux)
cargo install --path . --bin valkey-lab

# benchmark localhost, 60s, 80:20 GET:SET
valkey-lab

# custom target
valkey-lab -h 10.0.0.1 -c 16 -P 32

# find max throughput under p999 < 1ms
valkey-lab saturate --slo-p999 1ms

# save results, view in browser
valkey-lab --parquet results.parquet -d 5m
valkey-lab view results.parquet

Flags mirror valkey-benchmark conventions: -h, -p, -c, -P, -d, -t. Full reference in VALKEY-LAB.md.
