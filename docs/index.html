Why We Built This

Benchmarking is broken
for capacity planning.

Every production Valkey deployment has a throughput ceiling — the point where tail latency spikes and your SLOs break. Knowing that number before you ship is the difference between capacity planning and firefighting at 2 AM.

The built-in valkey-benchmark can't answer this question. It runs one command type at a time, has no rate control, and gives you a single max-throughput number that tells you nothing about what happens to latency at your actual production load.

To find out "what's the highest request rate where my p999 stays under 1 ms?", you'd need to run valkey-benchmark dozens of times, manually adjusting the rate limit each time, recording latency at each step, and hoping nothing drifts between runs. Nobody does this.

valkey-lab automates it in one command. It starts at a low request rate, ramps up by 5% each step, and stops after three consecutive SLO violations. The result is the maximum compliant throughput — the number you actually need for capacity planning.

Mixed workloads

Real traffic is GETs and SETs together. valkey-lab runs them concurrently at your chosen ratio so you measure how they actually contend.

Rate control

Set a target request rate and measure latency at that specific load — not just flat-out max throughput.

Saturation search

Define your latency SLO. valkey-lab finds the exact throughput ceiling automatically.

How It Works

Five phases.
Zero guesswork.

Every benchmark run follows the same disciplined sequence. Each phase is visible in the output so you always know where you are.

1. Precheck

Verify the server is reachable.

2. Prefill

Write every key for a warm cache.

3. Warmup

Let caches stabilize unrecorded.

4. Running

Record per-second metrics.

5. Results

Latency breakdowns and summary.

Saturation Search

Find the ceiling
automatically.

Define your latency SLO. valkey-lab starts at a low request rate, increases geometrically, and stops after three consecutive failures. The result is the maximum throughput that stays within your SLO. 

valkey-lab saturate --slo-p999 1ms -c 16 -P 32
valkey-lab saturate --slo-p999 1ms

Get Started

Install and run

# build from source (requires Linux)
cargo install --path . --bin valkey-lab

# benchmark localhost, 60s, 80:20 GET:SET
valkey-lab

# custom target
valkey-lab -h 10.0.0.1 -c 16 -P 32

# find max throughput under p999 < 1ms
valkey-lab saturate --slo-p999 1ms

# save results, view in browser
valkey-lab --parquet results.parquet -d 5m
valkey-lab view results.parquet

Flags mirror valkey-benchmark conventions: -h, -p, -c, -P, -d, -t. Full reference in VALKEY-LAB.md.
