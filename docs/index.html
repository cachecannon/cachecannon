The Problem

valkey-benchmark leaves gaps

The built-in valkey-benchmark is great for quick smoke tests, but falls short for production capacity planning.

🔀

No Mixed Workloads

valkey-benchmark runs one command type at a time. Real workloads are a mix of GETs and SETs that contend for CPU, memory, and network differently.

🚦

No Rate Limiting

Without rate control you always run at max throughput, which tells you nothing about latency at your actual production request rate.

🔍

No Saturation Search

Finding "the max throughput where p999 stays under 1 ms" requires dozens of manual runs. valkey-lab automates it in one command.

Saturation Search

Find your server's limit — automatically

One command ramps request rate until your latency SLO breaks, then reports the maximum compliant throughput.

bash
valkey-lab saturate --slo-p999 1ms -c 16 -P 32

Quick Start

Up and running in 60 seconds

Install
# Build from source (requires Linux with io_uring)
git clone https://github.com/cachecannon/cachecannon.git
cd cachecannon
cargo install --path . --bin valkey-lab
Basic Usage
# Benchmark localhost:6379 for 60s (defaults)
valkey-lab

# Custom host, 16 connections, pipeline depth 32
valkey-lab -h 10.0.0.1 -c 16 -P 32

# 30-second run, GET-only workload
valkey-lab -d 30s -r 100:0

# Find max throughput under p999 < 1ms SLO
valkey-lab saturate --slo-p999 1ms
